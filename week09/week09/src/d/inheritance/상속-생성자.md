# 상속에서의 생성자 호출 규칙
## 1. 기본 개념 : 생성자 호출 순서
* 객체가 생성될 때는 부모 클래스 -> 자식 클래스 순서로 초기화된다.
* 그래서 자식 클래스의 생성자는 **항상 부모 생성자를 먼저 호출**해야 한다.
* 이때 컴파일러가 자동으로 `super()`(인자 없는 부모 생성자 호출)을 첫 줄에 삽입한다.
```java
class Animal {}

class Dog extends Animal {
	String breed;
	
	Dog() {
		super(); // 컴파일러가 자동으로 넣어줌
	}
}
```

## 2. 기본 생성자 (default constructor)
* 클래스에 **아무 생성자도 정의하지 않으면** 자바는 자동으로 기본 생성자를 만들어준다.
```java
class Animal {
	// 컴파일 시 자동 생성됨
	Animal() {}
}
```

## 3. 부모 클래스에 생성자를 직접 정의할 때
```java
class Animal {
	String name;
	int age;

	// 생성자 직접 정의
	Animal(String name, int age) {
		this.name = name;
		this.age = age;
	}

}


class Dog extends Animal {
	String breed;
	
	Dog() {
		super(); // 에러
	}
}
```
* 위처럼 생성자를 직접 정의하면 기본 생성자(Animal())는 자동으로 생성되지 않는다.

* 이 때 자식 클래스 생성자 내부에 자동으로 `super()`가 삽입되는데 부모 클래스에 `Animal()` 이 없기 때문에 컴파일 에러가 발생한다.

### 방법 1 : 부모 생성자에 맞게 `super(인자...)` 직접 호출
```java
class Dog extends Animal {
	String breed;

	Dog(String name, int age, String breed) {
		super(name, age); // 부모 생성자 직접 호출
		this.breed = breed;
	}

}
```

### 방법 2 : 부모 클래스에 기본 생성자 추가
```java
class Animal {
	Animal() {} // 기본 생성자 추가
	Animal(String name, int age) {...}
}
```

> 부모가 직접 생성자를 만들면 자바는 기본 생성자를 안 만들어준다.   
> 그래서 자식이 부모 생성자를 직접 불러야 한다.
