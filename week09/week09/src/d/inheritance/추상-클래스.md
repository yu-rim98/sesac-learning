## 개념
> **추상 클래스(abstract class)** 란 공통 속성과 기능은 정의하지만 **구체적인 동작은 자식 클래스에게 맡기기 위한 설계도 클래스**이다.

* 완성되지 않은 클래스 (미완성 설계도)
* `abstract` 키워드로 선언
* 인스턴스 생성 불가(`new` 사용 불가)
* 자식 클래스가 상속받아 **미완성(추상) 메서드를 반드시 구현해야 한다.**

---
## 추상 메서드(Abstract Method)
```java
abstract void makeSound();
```
* 구현부 `{}`없이 선언만 있는 메서드
* 해당 메서드는 꼭 필요하지만 구체적인 동작은 자식이 알아서 정의해야 한다는 의미를 갖는다.
* 자식 클래스에서 **반드시 오버라이딩 해야 한다.**

---
## 사용 목적
1. 공통 기능의 통일
    * 모든 하위 클래스가 동일한 이름/형태의 메서드를 갖게한다.
    * `makeSound()`, `getArea()`
2. 코드 재사용
    * 공통 필드나 구현된 메서드는 부모(추상 클래스)에서 정의 후 상속받아 사용한다.
3. 설계 강제
    * 자식 클래스가 반드시 구현해야 할 기능을 명확하게 강제한다.
    * `abstract` 메서드
4. 객체 생성 제한
    * 해당 클래스 자체는 설계용이므로 인스턴스로 만들면 안된다는 의도를 코드로 표현할 수 있다.

```java
abstract class Animal {
	String name;

	public Animal(String name) {
		this.name = name;
	}

	void sleep() {
		System.out.println("잠자기");
	}

	abstract void makeSound(); // 추상 메서드

}

class Dog extends Animal {
	public Dog(String name) {
		super(name);
	}

	@Override
	void makeSound() {
		System.out.println("멍멍");
	}

}
```
* `Animal`은 **공통 속성(name)과 공통 로직(sleep)** 을 정의하고, `makeSound()`는 각 동물이 알아서 구현하도록 미뤄둔 상태이다.

## 추상 클래스 vs 일반 클래스 vs 인터페이스
| 구분      | 일반 클래스             | 추상 클래스             | 인터페이스                                            |
| ------- | ------------------ | ------------------ | ------------------------------------------------ |
| 인스턴스 생성 | 가능                 | 불가능                | 불가능                                              |
| 목적      | 구체적 기능 구현          | 설계 + 일부 구현         | 순수 설계 명세                                         |
| 메서드 구현  | 모두 구현              | 일부만 구현 가능          | (Java 8 전) 전부 미구현, (Java 8 이후) default/static 가능 |
| 상속/구현   | `extends` (1개만 가능) | `extends` (1개만 가능) | `implements` (여러 개 가능)                           |

---
## 정리
* `abstract` 클래스는 **공통 설계의 기반**이며 인스턴스를 생성할 수 없다.
* `abstract` 메서드는 **구현을 강제**하며 자식이 반드시 오버라이드해야 한다.
* 즉, 부모는 설계, 자식은 구현하는 구조
* 추상화의 목적은 공통화와 통일성 확보이다.

> 즉, 추상 클래스는 객체를 만들기 위한 것이 아니라 **공통 규칙(명세)을 정의하고 자식 클래스가 구현하도록 강제하기 위한 설계 도구**이다.
